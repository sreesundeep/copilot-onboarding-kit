# Contributing to Copilot Onboarding Kit

Thank you for considering contributing! This project is open to everyone — whether you're fixing a typo, adding a new prompt, or improving a diagram.

## 🤝 How to Contribute

### Quick Contributions
- **Fix a typo or improve wording** — submit a PR directly
- **Add a new prompt** — add it to `5-REUSABLE-PROMPTS.md` under the right category
- **Improve a diagram** — update the Mermaid code in any guide file

### Larger Contributions
1. **Open an Issue** first to discuss your idea
2. **Fork** the repository
3. **Create a branch** (`git checkout -b feature/my-improvement`)
4. **Make your changes**
5. **Submit a Pull Request** with a clear description

## 📝 Guidelines

### Content Style
- Keep prompts **generic** — they should work on any codebase, any language
- Use **clear, simple English** — this kit is for developers worldwide
- Always include **Mermaid diagram examples** where it makes sense
- Use `[PLACEHOLDERS]` in prompts for project-specific values

### File Naming
- Guide files follow the pattern: `N-KEBAB-CASE-NAME.md`
- Keep the numbered sequence intact

### What We're Looking For
- ✅ New prompts that help understand codebases faster
- ✅ Better Mermaid diagram templates
- ✅ Translations to other languages
- ✅ Tips for specific tech stacks (React, .NET, Python, etc.)
- ✅ Real-world examples of the kit in action
- ✅ Integration guides (VS Code, JetBrains, Neovim)

### What We're NOT Looking For
- ❌ Project-specific content (keep it generic)
- ❌ Prompts that only work with one language/framework
- ❌ Proprietary tool recommendations

## 🧪 Testing Your Changes

Since this is a documentation project, "testing" means:
1. Verify Mermaid diagrams render correctly (use [Mermaid Live Editor](https://mermaid.live))
2. Check all internal links work
3. Try your prompts on a real codebase to confirm they produce useful output

## 📜 Code of Conduct

Be kind, be respectful, be constructive. We're all here to help developers onboard faster.

## 💬 Questions?

Open an issue with the `question` label — we're happy to help!
